# Complete Prompt for Netflix-Style User Panel (OTT Platform)


Create a fully functional, professional OTT streaming platform user panel in a single HTML file with the following specifications:


## Design Philosophy


### Premium Modern Aesthetic
- **Dark Theme**: Deep blacks (#0a0a0a) with rich accent colors
- **Glassmorphism Effects**: Subtle blur and transparency
- **Smooth Animations**: Micro-interactions throughout
- **Layout Inspiration**: Netflix/Disney+ inspired with modern touches
- **Gradient Accents**: Purple to Blue (#8b5cf6 → #3b82f6)
- **Typography**: Inter or Poppins font family
- **Spacing System**: Consistent 8px spacing
- **Hover Effects**: Scale (1.05x) and glow effects
- **Loading States**: Skeleton loading for premium feel


### Color Scheme
- Background: #0a0a0a (Deep Black)
- Surface: #1a1a1a (Dark Gray)
- Surface Elevated: #2a2a2a (Lighter Gray)
- Primary Accent: #8b5cf6 (Purple)
- Secondary Accent: #3b82f6 (Blue)
- Success: #10b981 (Green)
- Warning: #f59e0b (Amber)
- Error: #ef4444 (Red)
- Text Primary: #ffffff (White)
- Text Secondary: #a3a3a3 (Gray)


### Gradients & Effects
- Hero Overlay: linear-gradient(to top, #0a0a0a 0%, transparent 100%)
- Card Hover: linear-gradient(135deg, #8b5cf6 0%, #3b82f6 100%)
- Button Primary: linear-gradient(135deg, #8b5cf6 0%, #7c3aed 100%)
- Glassmorphism: backdrop-filter: blur(10px); background: rgba(26, 26, 26, 0.7)
- Shadow: 0 8px 32px rgba(0, 0, 0, 0.4)
- Glow: 0 0 20px rgba(139, 92, 246, 0.5)


## Technical Stack


### Firebase Integration
- **Firebase Authentication**: User login/registration
- **Firestore Database**: All data storage
- **URL-Based Images**: All images via URLs (NO Firebase Storage)
- **Real-time Updates**: Live data synchronization


### Performance Features
- Lazy loading for images
- Skeleton screens during load
- Debounced search (300ms)
- Optimized Firebase queries
- Client-side caching
- Progressive image loading
- Infinite scroll optimization


### Responsive Design
- Mobile-first approach
- Mobile: < 768px
- Touch-friendly interface

### 36. Content Loading Strategy


**Initial Page Load:**
1. Load critical CSS inline
2. Load Firebase SDK
3. Check authentication state
4. Load user preferences from localStorage
5. Fetch hero carousel data (5 items)
6. Fetch first 3 content rows (10 items each)
7. Show skeleton loaders for rest


**Infinite Scroll:**
- Load more when user reaches 80% of scroll
- Load 10-20 items per batch
- Show loading indicator at bottom
- Handle end of content gracefully


**Image Loading:**
- Use Intersection Observer
- Load images when 200px from viewport
- Blur placeholder while loading
- Fade in when loaded


**Data Caching:**
- Cache fetched movies for 1 hour (localStorage)
- Cache user preferences indefinitely
- Cache categories indefinitely
- Clear cache on logout
